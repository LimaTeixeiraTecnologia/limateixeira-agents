# Prompt_Engineering_Standard.md

# PROMPT ENGINEERING STANDARD

Versão: 1.0

Status: Documento Oficial

Classificação: Engenharia Corporativa

---

# OBJETIVO

Definir o padrão oficial para construção de prompts utilizados por todos os Diretores Virtuais.

---

# PRINCÍPIOS

- Clareza.
- Objetividade.
- Modularidade.
- Reutilização.
- Menor consumo possível de tokens.
- Uso prioritário de documentação.
- Evitar redundâncias.

---

# ESTRUTURA PADRÃO

Todo prompt deverá conter:

1. Papel.
2. Objetivo.
3. Contexto.
4. Restrições.
5. Processo de decisão.
6. Critérios de qualidade.
7. Formato esperado.

---

# HIERARQUIA DE CONTEXTO

Prioridade obrigatória:

1. Knowledge.
2. Capabilities.
3. Workflow.
4. Dados da tarefa.
5. Conversa atual.

Nunca repetir conhecimento já existente na documentação.

---

# USO DE RAG

Consultar documentação somente quando necessária.

Carregar apenas os documentos relevantes para a tarefa.

Evitar contexto excessivo.

---

# ECONOMIA DE TOKENS

- Reutilizar documentos.
- Evitar prompts monolíticos.
- Preferir composição por Capabilities.
- Não duplicar instruções.

---

# SAÍDAS

Toda resposta deverá:

- Atender ao objetivo.
- Ser consistente com a documentação.
- Ser verificável.
- Evitar alucinações.
- Declarar limitações quando necessário.

---

# BOAS PRÁTICAS

- Uma responsabilidade por prompt.
- Linguagem objetiva.
- Evitar ambiguidades.
- Priorizar decisões reproduzíveis.

---

# PRINCÍPIO FINAL

Prompts devem coordenar inteligência, nunca substituir conhecimento estruturado.
