# REPORT GENERATION WORKFLOW

Versão: 1.0

Status: Documento Oficial

Classificação: Core Workflow Corporativo

---

# OBJETIVO

Gerar relatórios padronizados, consistentes e auditáveis para apoiar a tomada de decisão dos fundadores e dos Diretores Virtuais.

Este Workflow é reutilizável por qualquer área da empresa.

---

# RESULTADO ESPERADO

Ao final da execução deverá existir um relatório estruturado contendo:

- Resumo executivo.
- Indicadores.
- Principais resultados.
- Desvios.
- Riscos.
- Recomendações.
- Próximas ações.

---

# GATILHOS

- Agendamento periódico.
- Solicitação manual.
- Encerramento de Workflow.
- Evento relevante.

---

# ENTRADAS

- Dados consolidados.
- KPIs.
- Período analisado.
- Contexto.
- Objetivo do relatório.

---

# CAPABILITIES

- Analytics & Insights
- Storytelling

---

# OPERATORS

- PostgreSQL Operator
- LLM Operator
- Google Drive Operator

---

# PROCESSO

## Etapa 1

Coletar dados.

## Etapa 2

Validar consistência.

## Etapa 3

Consolidar indicadores.

## Etapa 4

Gerar resumo executivo.

## Etapa 5

Identificar riscos e oportunidades.

## Etapa 6

Produzir recomendações.

## Etapa 7

Versionar relatório.

## Etapa 8

Disponibilizar para consumo.

---

# REGRAS

- Utilizar apenas dados confiáveis.
- Declarar limitações quando existirem.
- Manter rastreabilidade das fontes.
- Padronizar estrutura dos relatórios.

---

# SAÍDAS

- Relatório.
- Dashboard (quando aplicável).
- Recomendações.
- Evidências.
- Histórico.

---

# CRITÉRIOS DE QUALIDADE

- Clareza.
- Objetividade.
- Precisão.
- Consistência.
- Valor para decisão.

---

# KPIs

- Tempo de geração.
- Precisão dos dados.
- Relatórios entregues.
- Taxa de utilização.

---

# PRINCÍPIO FINAL

Relatórios existem para apoiar decisões.

Toda informação apresentada deve ser confiável, objetiva e acionável.
